GPU memory bandwidth is one of the most common bottlenecks in high-performance computing. To write efficient execution code, we must look below the framework abstraction layers and understand the physical GPU memory hierarchy.

At the top of the hierarchy are **Registers**. Each thread has access to its own local registers, which operate at near-zero latency. However, register space is extremely limited; exceeding it causes "register spilling" to global memory (DRAM), severely dragging down throughput.

Next is **Shared Memory**. This is a user-managed cache shared by all threads within a block. It is located on-chip and is incredibly fast, but accesses must be carefully designed to avoid bank conflicts. The hardware divides shared memory into 32 banks; if multiple threads in a warp access different locations in the same bank simultaneously, the requests are serialized.

Global Memory is the largest memory space on the GPU but also the slowest, requiring hundreds of clock cycles for access. To mitigate this latency, global memory requests are coalesced. If threads within a warp access contiguous addresses, the hardware satisfies the requests in a single transaction.

Here is a simple example of a tiled matrix multiplication kernel loading data into shared memory to avoid redundant global memory reads:

```cpp
__global__ void matrixMulTiled(float *C, float *A, float *B, int N) {
    __shared__ float s_A[TILE_WIDTH][TILE_WIDTH];
    __shared__ float s_B[TILE_WIDTH][TILE_WIDTH];

    int tx = threadIdx.x; int ty = threadIdx.y;
    int row = blockIdx.y * TILE_WIDTH + ty;
    int col = blockIdx.x * TILE_WIDTH + tx;
    float value = 0;

    for (int ph = 0; ph < N/TILE_WIDTH; ++ph) {
        s_A[ty][tx] = A[row * N + ph * TILE_WIDTH + tx];
        s_B[ty][tx] = B[(ph * TILE_WIDTH + ty) * N + col];
        __syncthreads();

        for (int k = 0; k < TILE_WIDTH; ++k) {
            value += s_A[ty][k] * s_B[k][tx];
        }
        __syncthreads();
    }
    C[row * N + col] = value;
}
```

Understanding these memory hierarchies is essential for anyone trying to push the boundaries of hardware performance.
